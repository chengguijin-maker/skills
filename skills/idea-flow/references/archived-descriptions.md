# 旧技能原始 description 备份

恢复时把对应内容复制回各自 SKILL.md 的 description 字段即可。

## idea-ops-lite 原始 description

```
单文件优先的轻量化 idea→验证工作流。把零散想法、模糊需求、实验性功能快速收敛成一个 WORK.md 主控文件 + AGENTS.md 约束入口 + 最小仓库结构。覆盖：调研登记与去重、目标与非目标界定、架构边界、实施计划、验证点、中间材料生命周期管理。适用于 git 仓库、本地 worktree、离线文档驱动场景。当用户说"整理一下这个想法""帮我建个项目骨架""收敛一下需求""把散乱文档归拢""初始化一个轻量项目""接管这个仓库""先规划不要写代码"时，都应该使用这个 skill。即使用户没有明确提到 idea-ops-lite，只要任务涉及从想法到可执行方案的收敛整理，就应该触发。
```

## idea-spec-execution 原始 description

```
将模糊产品或功能想法快速整理为问题卡、轻量规格、任务拆解和按需执行编排的中文工作流。用于 idea→需求、想法转 spec、轻量 PRD/设计、需求边界澄清、任务拆解、实现计划、并行执行计划等请求；当用户明确不想走重型阶段文档，只想快速收口为可执行材料时使用。
```
