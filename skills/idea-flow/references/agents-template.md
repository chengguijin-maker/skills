# AGENTS

## 工作规则

- 单文件优先
- 主控文件是WORK.md
- 先问题后方案，先定锚再收口
- 先写非目标
- 先查旧调研
- 结论回填主文件
- 临时材料进scratch
- 有价值再归archive
- 吸收后删除草稿
- 一任务一worktree
- 小步频繁提交
- 验证点必须落表
- 风险必须显式写
- 决策写入ADR-lite
- 状态持续更新

## 方法论规则

- Constraints 与 Assumptions 分开写
- V1 In/Out 必须明确
- 组件/模块 ≤ 9
- 按 demoable sprint 切片，不按技术层分组
- 每个任务写清 Validation 和 Done Signal
- "顺手做"类内容放进 deferred
- 只对高影响分歧做方案比选

## 去重规则

- 同题先查RES表
- 同题优先更新原行
- 同源优先复用摘要
- 冲突结论必须标记
- 低价值材料不留

## 拆分规则

- 非必要不拆文件
- 拆分后主文件保索引
- 子文件顶部写回链
- 子文件继承项目UID
- 必须说明拆分原因

## worktree 规则

- 禁止直改主分支
- 分支只做一个目标
- 验证后尽快合并
- 合并后删除worktree

## 归档规则

- 可复查才归档
- 高成本材料可归档
- 决策证据可归档
- 已吸收材料不归档
- 过期草稿直接删除
