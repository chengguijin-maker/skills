# 强约束提示模板

## 1. 通用连续执行模板

```text
使用 $autonomous-task-executor 处理这个任务。
如果我已经给出明确目标、范围、计划、任务清单或验收标准，你必须将其视为执行合同，而不是讨论草案。
默认连续执行直到任务完成，不要在每一步后停下来问“下一步做什么”。

只有在以下情况才允许停下提问：
1) 需求冲突或目标不明确到无法继续；
2) 缺少必要权限、凭据、文件、环境变量或外部输入；
3) 即将进行不可逆、破坏性或高风险操作；
4) 继续执行有较高概率明显偏离我的真实意图。

除以上情况外：
- 自行做最保守合理的假设并继续；
- 自动推进后续步骤，不等待我逐步确认；
- 只做简短进展同步，不把同步变成征求下一步；
- 优先执行最短、最有效的验证；
- 所有命令加 timeout，并确认程序有响应没卡住；
- 最终一次性输出：完成情况、主要改动、验证结果、关键假设、剩余风险、可选下一步。
```

## 2. 代码开发模板

```text
使用 $autonomous-task-executor 完成这项开发任务。
已给出的需求、计划和验收标准都视为已确认，请直接连续执行到完成。
不要在实现、改文件、补测试、跑验证之间反复问我要不要继续。

你的默认动作是：
1) 按计划实现；
2) 做必要的最小改动；
3) 运行短小快速的针对性验证；
4) 修复显性问题；
5) 给出结构化交付结果。

只有在真实阻塞时才停下：权限不足、必需输入缺失、需求冲突、不可逆高风险操作。
其余情况一律先做保守假设并继续。
```

## 3. 修 Bug 模板

```text
使用 $autonomous-task-executor 修复这个问题。
默认流程是：快速定位 -> 最小修复 -> 针对性验证 -> 汇总结果。
不要在中途反复征求下一步，除非遇到真实阻塞。

要求：
- 不扩大范围；
- 不顺手重构无关代码；
- 不用降低测试标准来换取通过；
- 所有命令加 timeout；
- 最终明确说明根因、修改点、验证结果、剩余风险。
```

## 4. 强压制回问模板

```text
从现在开始，除非遇到真实阻塞，否则禁止停下来问我“下一步做什么”“是否继续”“是否要我再做 X”。
已有计划就继续执行，没有显式计划就自行补全最短计划后继续执行。
把中间的不确定性优先转化为保守假设，而不是转化为问题抛回给我。

允许停下的条件仅限：
- 缺少必要权限、凭据或外部输入；
- 需求存在明显冲突；
- 即将进行不可逆或高风险操作；
- 继续执行会大概率做错方向。

如果没有触发以上条件，就持续推进直到交付闭环完成。
```

## 5. 轻量版一句话模板

```text
使用 $autonomous-task-executor 按现有目标和计划连续执行到完成；除非真实阻塞，否则不要停下来问下一步，自己做保守假设并在最后统一汇报。
```

## 使用建议

- 需求已经很清楚时，优先用“通用连续执行模板”或“轻量版一句话模板”。
- 代码改动较多时，优先用“代码开发模板”。
- 明显是缺陷修复时，优先用“修 Bug 模板”。
- 某个环境下经常中途回问时，直接使用“强压制回问模板”。
