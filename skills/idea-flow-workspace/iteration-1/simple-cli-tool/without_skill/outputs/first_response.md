## 项目概述

一个命令行工具，能够将 Markdown 文件批量转换为 PDF，并支持用户自定义样式。

## 核心功能

1. **Markdown 转 PDF** — 将单个或多个 `.md` 文件转换为对应的 `.pdf` 文件
2. **批量处理** — 支持指定目录、glob 模式（如 `*.md`、`docs/**/*.md`）批量转换
3. **自定义样式** — 支持通过外部 CSS 文件或内置主题来控制 PDF 的排版和样式

## 需要进一步明确的问题

在动手之前，有几个方向性问题值得先想清楚：

### 技术选型
- **语言选择**：你倾向用哪种语言？常见选择有：
  - **Node.js/TypeScript** — 生态丰富，有 `markdown-it` + `puppeteer` 等成熟方案
  - **Python** — 有 `markdown` + `weasyprint` 等库
  - **Go/Rust** — 编译为单二进制文件，分发方便
- **转换引擎**：
  - 基于浏览器渲染（Puppeteer/Playwright）— 样式兼容性好，但体积较大
  - 基于专用排版引擎（WeasyPrint、wkhtmltopdf、Typst）— 更轻量

### 样式系统
- 是否提供几套内置主题（如 GitHub 风格、学术论文风格）？
- 自定义样式的粒度：仅支持 CSS，还是也支持页眉页脚、页码、封面等配置？
- 是否需要支持代码高亮主题的切换？

### 输入输出
- 是否需要支持 frontmatter（YAML 头部）来为每个文件单独指定标题、作者等元信息？
- 输出文件的命名和目录结构规则是什么？（原地生成、统一输出目录等）
- 是否需要将多个 Markdown 合并为一个 PDF？

### 使用方式
- 基本用法大致是这样吗？
  ```bash
  md2pdf ./docs/**/*.md --style ./my-style.css --output ./pdfs/
  md2pdf README.md --theme github
  ```
- 是否需要配置文件（如 `md2pdf.config.json`）来保存常用设置？
- 是否考虑 watch 模式（文件变化时自动重新生成）？

### 发布方式
- 发布到 npm / PyPI / Homebrew？
- 是否需要提供 Docker 镜像？

## 建议的下一步

1. **确定技术栈** — 先选定语言和转换引擎
2. **定义 MVP 范围** — 第一版只做核心功能：单文件/批量转换 + CSS 样式支持
3. **设计 CLI 接口** — 确定命令名、参数、选项
4. **搭建项目骨架** — 初始化项目、配置构建工具和测试框架
5. **迭代开发** — 从最简单的单文件转换开始，逐步增加批量处理和样式功能
