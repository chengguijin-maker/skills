# 数据库选型调研报告

## 调研目的

为 RustNote CLI 笔记工具选择合适的嵌入式存储方案。

## 候选方案对比

### SQLite

| 维度 | 评价 |
|------|------|
| 成熟度 | 极高，数十年历史 |
| 全文搜索 | FTS5 原生支持 |
| 部署 | 单文件，零配置 |
| Rust 生态 | rusqlite 库成熟 |
| 写并发 | 差（但单用户场景无影响） |
| 查询能力 | SQL 标准查询，灵活 |

### LevelDB

| 维度 | 评价 |
|------|------|
| 成熟度 | 高 |
| 全文搜索 | 不支持，需自行实现 |
| 部署 | 嵌入式 |
| Rust 生态 | 有绑定库 |
| 写入性能 | 优秀 |
| 查询能力 | 仅 key-value，无 SQL |

### Sled

| 维度 | 评价 |
|------|------|
| 成熟度 | 低 |
| 全文搜索 | 不支持 |
| 部署 | 嵌入式 |
| Rust 生态 | 原生 Rust 实现 |
| API 稳定性 | 不稳定，社区小 |
| 查询能力 | 有限 |

## 结论

**选择 SQLite** 作为存储方案。

核心理由：
1. FTS5 全文搜索是本项目的关键需求，SQLite 原生支持
2. SQL 查询灵活，标签多对多关系表达自然
3. 单文件部署，用户体验好
4. 单用户 CLI 场景，写并发弱点不构成问题
